# Generation coherence & recovery — session notes

Context for why the API generation path was reworked toward a recoverable single-pass
"coherent author," and notes on the open multi-provider question. Companion to
`docs/ai-initial-generation-pipeline.md` (the prompt/contract) — this file is the
*rationale and the change log*.

---

## 1. The observation that started it

A deck generated by **Claude Code** (prompt-export mode, `HTML_PPT_AI_MOCK=1`) looked
clearly better than the same deck generated through the **API** — even though both use
the **same Opus 4.8 model**.

**Root cause: the harness, not the model.** Claude Code wins because of *how* it runs the
job, not which model runs it:

- **Single coherent author.** It writes the whole deck (`index.html` + `style.css` +
  `script.js`) in one context, so palette, animation rhythm, and "different component per
  slide" stay consistent.
- **Incremental & recoverable.** It writes files one at a time; a stall never destroys the
  whole result.
- **Reference-grounded.** It can actually `Read` `sample_deck/` and match the quality bar.
- **Verify-and-fix loop.** It checks slide counts, fixes invalid CSS, scans for banned
  words in-loop.
- **Codebase-grounded content.** For "DeckFlip about itself," the real architecture/numbers
  were in context — accurate content reads as higher quality.

The API path, by contrast, ran the job as **bounded, cached, parallelized,
deterministically-assembled fragments** — optimized for cost and robustness at scale, which
trades away single-author polish.

---

## 2. Why `single` had been abandoned for `multi`

The real reason wasn't just "truncation gives a shorter deck." It was worse:

> single-pass sometimes **consumed a lot of tokens and returned nothing** — the worst case.

Because single-pass output went through a **structured-output (`json_schema`) tool call**, a
response that ran long died **mid-JSON** and was **unparseable** → `coerceDeckFiles` got
nothing. You paid the full ~32K output tokens for **zero usable bytes**. Switching the
default to `multi` (bounded per-call output) was the correct call *for reliability*, but it
gave up the single-author coherence.

**Key reframe:** the catastrophic failure is a property of *"one atomic structured-output
call,"* **not** of *"one coherent author."* Those are separable. Claude Code is a single
author that *also* never fails catastrophically, because it's incremental and recoverable.
The fix = port that recoverability to the API single-pass.

---

## 3. What was implemented (applied this session)

Goal: keep single-pass coherence, remove the total-loss failure, add reference + verification.

### Provider layer — `api/_generation/providers.ts`
- **Extended output header** — `anthropicHeaders()` sends an optional `anthropic-beta`
  (`HTML_PPT_ANTHROPIC_BETA`) so a 64K+ output budget is honored where supported. Off by
  default (an unsupported value would 400 the call).
- **Streamed free-text path** — new `callAnthropicRaw` (SSE-parsed) + `callOpenAiRaw`
  fallback behind `callRawWithFallback`. Streaming avoids request-timeout on long decks and
  captures partial output. Toggle: `HTML_PPT_GEN_STREAM` (default on).
- **Continuation support** — `RawCall.assistantPrefix` resumes a truncated assistant turn
  (trailing whitespace trimmed, which Anthropic requires).
- **Refactor** — retry/fallback hoisted into generic `fallbackOver`/`retryOver` so the
  structured (`callWithFallback`) and free-text (`callRawWithFallback`) paths share identical
  retry/logging.

### Parser — `shared/generation/schemas.ts`
- **`parseDelimitedDeck()`** — parses `===FILE: …===` / `===BRIEF===` / `===END===` blocks,
  keeps every **completed** file, repairs a **truncated** trailing `index.html` (closes
  dangling `<section>`/`</main>`/`</html>`), and reports `complete` (END sentinel seen).
  Verified: a mid-stream cut still yields intact CSS/JS + a renderable partial deck.

### Prompt — `api/_generation/prompt.ts`
- Single-pass contract rewritten to the **delimited output format**, with **`index.html`
  written last** so truncation drops tail slides, not everything.
- **`SAMPLE_DECK_EXCERPT`** — a real trimmed slice of `sample_deck/Sample_KoreanPPT` (tokens
  + cover/divider/body structures + IO wiring) embedded in the cached prefix, so the model
  *sees* the reference like Claude Code does.

### Orchestration — `api/_generation/generate.ts`
- `generateSinglePass` rewritten: streamed raw call → **continuation loop**
  (`HTML_PPT_SINGLE_MAX_CONTINUATIONS=4`) until the END sentinel → `parseDelimitedDeck` →
  existing lint-gated QA + overflow guard.
- **Default mode flipped to `single`** (was `multi`). Fallback ladder unchanged:
  `single → multi → mock`.

### Lint — `api/_generation/assemble.ts`
- `deckLint` strengthened with the checks previously done by hand: **canvas ↔ `__chartInit`
  parity**, **too-few-slides** (truncation signal), and **banned filler words**
  (시사점/패러다임/재정의/진정성). These feed the QA repair pass.

### Config — `.env` / `.env.example`
```
HTML_PPT_GEN_MODE=single
HTML_PPT_SINGLE_MAX_TOKENS=64000
HTML_PPT_SINGLE_MAX_CONTINUATIONS=4
HTML_PPT_GEN_STREAM=1
HTML_PPT_QA_MODE=always
# HTML_PPT_ANTHROPIC_BETA=output-128k-2025-02-19   # enable only after verifying your model accepts it
```

### Caveats
- **`HTML_PPT_AI_MOCK` must be `0`** to exercise this path; `=1` is prompt-export (Claude
  Code) mode and short-circuits before any API call. Restart `npm run dev` after changing env.
- **`HTML_PPT_ANTHROPIC_BETA` stays unset** until the exact extended-output beta id is
  verified for the pinned `claude-opus-4-8`. If 64K is rejected/capped, set it; if Opus
  allows 64K natively, leave it off.
- Typecheck passes; parser salvage verified via a throwaway tsx test.

## 4. Open question — multi-provider / multi-agent (NOT done)

Idea: route sub-tasks to different models by strength — e.g. **Gemini** for Korean copy,
**GPT** for planning logic, **Claude** for design/CSS.

### The tension
This is the **opposite** of the single-coherent-author win. Splitting work back across
providers re-introduces fragmentation **plus** a new failure: cross-model **voice/aesthetic
clash**. It belongs on the **multi-pass** pipeline (which is already decomposed), not
single-pass (which by definition can't be split).

### Natural role → pass mapping
| Pass | Task | Hypothesized best model |
|---|---|---|
| `emit_plan` | outline + logic + web search | GPT |
| `emit_foundation` | style.css / script.js | Claude |
| `emit_slides` | markup rendering | Claude |
| **new** `emit_copy` | Korean copy polish (text nodes only) | Gemini |
| `emit_qa` | verify/repair (integration glue) | Claude |

"Korean copy" is **not** a clean slice of `emit_slides` (which emits markup + copy together)
— the clean seam is a dedicated **text-node-only copy-editor pass** (same discipline as the
editor's `setEditableText`), so language never rewrites layout.

### What it would take
1. **Provider registry + Gemini adapter** — generalize the `anthropic|openai` union; add
   `callGemini`/`callGeminiRaw` (`generativelanguage.googleapis.com/.../generateContent`).
2. **Per-pass routing** — `callWithFallback(providers, …)` already takes a `providers` arg
   per call; add `resolveProvidersForRole(role)` + `HTML_PPT_{PLAN,FOUNDATION,COPY}_PROVIDER`.
3. **Schema translation** — Gemini's `responseSchema` is a stricter OpenAPI subset (no
   `additionalProperties`); needs a `toGeminiSchema()` cleaner. (Single-pass delimited output
   is free-text → no schema needed there.)
4. **Streaming parity** — Gemini SSE has a different event shape than the Anthropic reader.

### Costs / risks
- **Prompt-cache loss** — the big system prefix caches *per provider*; routing every pass to
  a different model means no cross-pass cache reuse (real cost bump on a large prompt × passes).
- **Voice incoherence** — mitigated by the threaded `DesignBrief` + `componentManifest`, not
  eliminated.
- **Latency + debugging** — 3 vendors = 3 rate limits, 3 failure modes, bigger fallback matrix.

### Recommendation (in order)
1. **Validate per-model superiority first** — A/B the same deck with each model in each role.
   Claude and GPT are both strong at Korean; the win may not exist. Don't pay the integration
   tax on a hunch.
2. **Start with the one low-risk seam** — keep the coherent build on one model, add a
   **Gemini text-node copy-editor pass** at the end (touches only language, can't break design).
3. **Only then** build the full provider matrix if step 1 proves the splits are worth it.
4. Keep **QA on the strongest model (Claude)** — with multi-agent it becomes the integration
   glue, not optional polish.

**Status: deferred — discussed, not implemented.**

---

## 5. TL;DR
- Same model, different harness → the gap was process, not Opus.
- Single-pass is now a **recoverable coherent author**: delimited output + streaming +
  continuation, so a truncation salvages instead of losing everything. It's the default again.
- Multi-provider routing is a real but separate feature that *trades coherence for
  specialization*; validate the per-model wins before building it.
